# IELTS 单词背诵工作流

## 核心规则（最高优先级）

用户输入一个纯数字编号（6~7 位，如 `260616` 或 `2606121`）时：
- **禁止提问、禁止确认、禁止征求意见**
- 直接执行全部步骤，像脚本一样自动化
- 首次使用前，确保 Claude Code 已授权文件读写和脚本运行权限

## 输入模式

### 仅数字编号（自选主题）
```
260711
```
→ AI 按你预设的主题方向**自选主题和 6~8 个中文短语**，新建 `words/<编号>.txt`，改写并生成音频。

> 编号长度灵活：`260711`（当天）、`2607111`（当天第二篇）、`2607112`（当天第三篇）都是独立文件，互不覆盖。

### 数字编号 + 中文短语（用户指定）
```
260711 色谱, 分离, 柱效, 流动相, 检测波长
```
→ 用你给的词汇，新建 `words/<编号>.txt`，改写并生成音频。

## 前置条件

```bash
pip install -r requirements.txt
```

## 步骤

### 1. 确保 txt 存在
- 模式 A：检查 `words/<编号>.txt` 是否存在
- 模式 B：**直接 Write 新建** `words/<编号>.txt`，内容为"1\t<中文短语>"（即原始词汇行）

### 2. 改写 txt（覆盖原文件）
按以下固定 5 节排版**直接 Write 写入**，不询问是否覆盖：

```
原始词汇
==========
<保留用户原文>

单词翻译
==========
<中文 → 英文，雅思级别表达>

英文背诵段落
==========
<用上所有词写一段雅思水平的英文段落，复杂句，避免低级词汇>

中文对照
==========
<英文段落的中文翻译>

重点短语
==========
<从段落中抽出可迁移的搭配/句型，中英对照>
```

- 节标题固定为中文 + `==========` 分隔线，`tts.py` 靠这个正则抽取
- 英文段落约 120~180 词，句子要有层次感（从句、并列、插入语），贴近雅思写作 Task 2 风格
- 重点短语选 10~14 个可迁移搭配

### 3. 生成音频
```bash
python tts.py <编号>
```
- 默认英式发音 `en-GB-SoniaNeural`，语速 `-5%`
- 输出 `words/<编号>.mp3`
- 可选参数：`--voice` 指定发音人，`--rate` 指定语速

### 4. 验证
```bash
ls -lh words/<编号>.mp3
```

## 自定义主题方向

修改 `仅数字编号（自选主题）` 下的主题列表为你感兴趣的领域，AI 会自动轮换。

## 自定义 TTS 语音

编辑 `tts.py` 中的默认值，或运行时指定：
```bash
python tts.py 260716 --voice en-US-AriaNeural --rate -10%
```

常用语音：
| Voice | 风格 |
|---|---|
| `en-GB-SoniaNeural` | 英式女声（默认） |
| `en-GB-RyanNeural` | 英式男声 |
| `en-US-AriaNeural` | 美式女声 |
| `en-US-GuyNeural` | 美式男声 |

## 约束
- 不改动用户原始词语行
- 不改 `tts.py` 脚本
- 不新建额外的 txt/md 文件（模式 B 新建 `<编号>.txt` 是唯一例外）
- 全程零交互，不征求任何确认
